## 4. Visual Direction — "Floral Editorial" (girly + professional, explicitly anti "AI-slop")

Selected level: **Floral Editorial** (feminine and pretty, but senior and polished — never cutesy/childish/bubblegum).

- **Palette:** warm cream/blush paper (`#FBF4F1` / soft `#FDF4F7`–`#FBEFF2` gradients), dusty-rose ink accent (`#C9607A` / `#D17A95`), deep rose-plum text (`#2B1A1F` / `#3A2128`), muted mauve for secondary text (`#9A7782` / `#6B4A55`). Single rose accent family — NOT the cliché purple→blue or emerald/indigo AI gradient.
- **Typography:** high contrast — an elegant display serif for headings (e.g. Fraunces / Playfair Display / Cormorant) + a clean monospace for labels/section numbers (e.g. JetBrains Mono). A refined **italic serif** used sparingly for a "script-like" feminine touch on key phrases. "Beauty-editorial × engineer" feel.
- **Feminine motifs:** delicate **custom SVG petal/floral accents** as faint, low-opacity background/corner ornaments and section dividers. Soft rounded cards and pill buttons, gentle (not heavy) shadows. Motifs are subtle and tasteful — decoration, never clutter. **No emoji, no hearts/sparkles, no Comic-Sans-style fonts.**
- **Layout:** asymmetric editorial grid, thin rose rules, section numbers (01/02/03), generous whitespace, large type as hierarchy.
- **Motion:** subtle — scroll reveal, count-up on the hero metrics, hover line/petal draws, soft fades. A calm animated hero accent (e.g. slowly drifting petals or a soft gradient bloom). No flashy/heavy animation. Respect `prefers-reduced-motion`.
- **Texture:** faint grain + subtle blush wash so it doesn't read as a flat template.
- **Forbidden:** generic purple→blue gradients, emoji-heavy copy, bubblegum/childish styling, filler marketing fluff, lorem padding, generic AI-template look.

### Design-quality requirement (per user instruction)
Implementation MUST be done using the **ui-ux-pro-max** skill (and **frontend-design** skill) for high design quality — not default/quick HTML. The site must NOT look AI-generated. Apply those skills' metric-based rules: intentional typography scale, real spacing system, component discipline, hardware-accelerated motion, accessibility.

## 6. Technical Approach

- **Single file** `putu/index.html`. No build step. Opens directly in browser; deployable free to GitHub Pages / Netlify drag-drop. Lives in a `/putu/` subfolder so it coexists with Yafi's root `index.html`.
- Tailwind via CDN (`cdn.tailwindcss.com`) with inline config for the custom rose/blush palette + fonts.
- Google Fonts: one display serif + one monospace.
- Vanilla JS for: ID/EN language toggle (data-attribute or JS dictionary), count-up on scroll (IntersectionObserver), scroll-reveal, hero floral/gradient animation, card hover detail.
- **Bilingual:** all copy stored as `data-id` / `data-en` (or a JS dictionary keyed by language); toggle swaps text and persists choice in `localStorage`. Default: ID.
- Accessibility: semantic HTML, alt text, sufficient contrast (verify rose-on-cream meets WCAG AA for body text), `prefers-reduced-motion` disables heavy animation.
- Performance: defer non-critical JS; target fast first paint.

## 7. Components / Units (each independently understandable)

- **i18n module** — dictionary + toggle + localStorage. Input: lang key. Output: swapped text. No dependency on other modules.
- **count-up** — animates a number from 0 to target (or reveals a non-numeric proof like "CEO") when scrolled into view. Self-contained.
- **scroll-reveal** — adds reveal class on intersection. Generic, reusable.
- **hero-accent** — floral/gradient animation; respects reduced-motion; isolated, swappable without touching content.
- **proof-card / service-card** — presentational; data + role badge passed via markup. Hover detail self-contained.
- **petal-motif** — reusable inline SVG ornament component used as faint accents/dividers.
